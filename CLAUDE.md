## 基本ルール

1. **明示的な確認なしに、ファイルの外部転送・アップロード・公開を行わないこと。**
2. コード変更・ファイル追加の際は、意図と影響範囲を明確にしてから実行すること。
3. 不明点がある場合は推測で進めず、確認を取ること。

---

## プロジェクト概要

- **サイト名**: Aoki Estate（アオキ・エステート）
- **業態**: 架空の地域密着型不動産会社（賃貸+売買）
- **目的**: ポートフォリオ掲載用の架空クライアントワーク
- **技術**: Vanilla HTML/CSS/JS（フレームワーク不使用）
- **設計書**: `design-spec.md` に全仕様を記載。実装時は必ず参照すること。

---

## 制作信条（絶対遵守）

以下はすべてのサイト制作において守るべき不変のルールである:

1. **差別化と個性**: 「よくあるテンプレート」に見えないこと。独自のデザイン判断を随所に入れる。
2. **AI作成感の排除**: AIが自動生成したような均一的・無個性なデザインにしないこと。人間のデザイナーが丁寧に作り込んだように見えること。
3. **タイポグラフィへのこだわり**: フォント選び・サイズの緩急・行間・letter-spacing・余白のすべてに意図を持つこと。

---

## デザイン方針

- **モダン×ナチュラル**（アースカラー系）
- 見出し明朝体（Noto Serif JP）× 本文ゴシック（Noto Sans JP）
- 英語見出し: セリフ体（Cormorant Garamond）
- UI要素: Poppins
- 背景は明るい（ライト系）。ダーク背景はフッターとバナーセクションのみ。
- 装飾は最小限。余白とタイポの力で世界観を構築。
- 物件写真には統一的な CSS filter を適用し、トーンを揃える。
- カラーパレット・サイズ・余白等の詳細は design-spec.md セクション2〜4を参照。

## 画像の扱い

### 基本方針
- 本プロジェクトでは**すべてのビジュアル箇所に実画像を使用する**方針。
- 画像はあおきさんが ChatGPT（DALL-E）等で生成・用意する。
- 実装段階ではプレースホルダー（CSSグラデーション + テキストラベル）を配置。
- プレースホルダーには alt 属性やHTMLコメントで、ファイル名・推奨サイズ・用途を明示し、差し替えを容易にすること。

### Hero画像
- `assets/img/hero.jpg` を参照
- 画像未配置時のフォールバック: --color-bg-warm ベースのグラデーション
- キャッチコピー: `住まいに、物語を。`（確定）
- オーバーレイ: `rgba(250,250,247,0.55)`（明るい白系半透明、確定）

### 写真の統一トーン
- すべての物件写真・イメージ写真に統一 filter を適用:
  `filter: brightness(1.02) saturate(0.92) contrast(1.02);`

### 仮完成後の画像生成プロンプト出力
- サイト仮完成後、あおきさんが ChatGPT に投げるための画像生成プロンプト（用途・サイズ・比率・雰囲気キーワード）の一覧表を別途出力する。
- Claude Code は仮完成報告時に、各画像の用途・推奨サイズ・雰囲気のキーワードをまとめて報告すること。

## コーディング規約

### HTML
- セマンティックHTML5（header, nav, main, section, article, aside, footer）
- BEM的なクラス命名（例: `.hero__title`, `.property-card__price`）
- アクセシビリティ: スキップリンク、aria属性、role、alt属性
- lang="ja"

### CSS
- CSS変数を :root で定義し一貫して使用
- モバイルファースト（min-width メディアクエリ）
- 1ファイル（assets/css/style.css）に統合
- セクション単位でコメント区切り
- box-sizing: border-box をグローバル適用
- 物件写真に統一 filter を適用: `filter: brightness(1.02) saturate(0.92) contrast(1.02);`

### JavaScript
- Vanilla JS、IIFE で囲む
- 1ファイル（assets/js/main.js）に統合
- defer 属性で読み込み
- prefers-reduced-motion を尊重
- DOM操作は querySelector / querySelectorAll を使用
